Excel vs. Python: E-Commerce Sales Analysis

A comparative analysis of a 2023 e-commerce dataset using Excel Pivot Tables and Python (Pandas + Matplotlib).

Project Overview

This project analyzes a 2023 e-commerce dataset by answering the same business questions using two analytical workflows: Excel Pivot Tables and Python (Pandas + Matplotlib). Findings show that revenue is concentrated in Electronics, Home Garden, and Sports products and is driven primarily by customer retention rather than discounts, device choice, or delivery speed. These results suggest that long-term growth efforts should prioritize repeat purchasing behavior over short-term tactical optimizations. A comparison of how each tool affects analysis speed, reproducibility, scalability, and visualization is also provided.

Key Business Findings

  • Revenue concentration: Revenue is concentrated in Electronics, Home & Garden, and Sports.
  • Discounts: Discounts show little measurable impact on average order value.
  • Customer retention: Returning customers drive revenue primarily through higher purchase frequency.
  • Device usage: Mobile dominates order volume while desktop shows slightly higher average order value.
  • Delivery experience: Delivery speed shows minimal relationship with customer ratings.

Both tools produced consistent analytical conclusions. The primary differences emerged in workflow efficiency, automation potential, and long-term scalability.

Excel vs. Python: Workflow Comparison

  • Speed for ad-hoc analysis: Excel Pivot Tables allow rapid aggregation and exploration with minimal setup, making them ideal for quick business questions.
  • Automation and reproducibility: Python requires more setup but produces a fully reproducible workflow for data cleaning, aggregation, and visualization.
  • Scalability: Excel performs well for moderate datasets but may become cumbersome at scale. Python is well suited for larger datasets and integration into automated pipelines.
  • Visualization flexibility: Excel provides fast built-in charts, while Python libraries like Matplotlib offer deeper customization and control.
  • Practical takeaway: Excel is ideal for exploratory analysis and stakeholder reporting, while Python is better suited for scalable analytical workflows and repeatable data pipelines.
