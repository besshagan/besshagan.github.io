Excel vs. Python: E-Commerce Sales Analysis

A comparative analysis of a 2023 e-commerce dataset using Excel Pivot Tables and Python (Pandas + Matplotlib). The project evaluates how analytical tools influence workflow, scalability, and reproducibility while producing the same business insights.

Project Overview

This project analyzes a 2023 e-commerce dataset by answering the same business questions using two analytical workflows: Excel Pivot Tables and Python (Pandas + Matplotlib). The objective is to compare how each tool affects analysis speed, reproducibility, scalability, and visualization.

Key Business Findings

  • Revenue concentration: Revenue is concentrated in Electronics, Home & Garden, and Sports.
  • Discounts: Discounts show little measurable impact on average order value.
  • Customer retention: Returning customers drive revenue primarily through higher purchase frequency.
  • Device usage: Mobile dominates order volume while desktop shows slightly higher average order value.
  • Delivery experience: Delivery speed shows minimal relationship with customer ratings.

Both tools produced consistent analytical conclusions. The primary differences emerged in workflow efficiency, automation potential, and long-term scalability.

Excel vs. Python: Workflow Comparison

  • Speed for ad-hoc analysis: Excel Pivot Tables allow rapid aggregation and exploration with minimal setup, making them ideal for quick business questions.
  • Automation and reproducibility: Python requires more setup but produces a fully reproducible workflow where data cleaning, aggregation, and visualization can be rerun automatically.
  • Scalability: Excel performs well for moderate datasets but becomes cumbersome at scale. Python handles larger datasets and integrates naturally into automated pipelines.
  • Visualization flexibility: Excel provides fast built-in charts, while Python libraries like Matplotlib offer deeper customization and programmatic control.
  • Practical takeaway: Excel is ideal for exploratory analysis and stakeholder reporting, while Python is better suited for scalable analytical workflows and repeatable data pipelines.
